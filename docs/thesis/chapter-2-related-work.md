# 第 2 章 相关技术与研究现状

## 2.1 区块链与智能合约技术

区块链通过分布式账本、共识机制、密码学哈希和不可篡改记录，为多主体协作场景提供了共同维护状态的技术基础。在数据交易场景中，交易双方往往缺少长期信任关系，平台方又可能同时掌握数据目录、交易记录、授权记录和争议处理权。若所有关键状态都由中心化平台维护，买卖双方难以独立验证订单是否被篡改、授权是否按约定发放、评价和争议记录是否被完整保留。区块链把交易过程转化为可验证的账本状态，能够缓解中心化平台带来的信任集中问题。

智能合约是在区块链网络中执行的业务逻辑。它可以把数据发布、订单创建、押金锁定、访问授权、交付确认、评价提交和争议处理等流程固化为状态机，并由网络共同验证执行结果。对于数据交易而言，智能合约的价值不在于保存大规模原始数据，而在于保存可审计的交易摘要、授权摘要和证据摘要。当买卖双方发生争议时，合约可根据链上状态和证据哈希执行退款、结算或扣罚，减少人工裁决的不透明性。

现有基于区块链的数据交易研究多强调确权、溯源和结算，但在 IoT 批次数据场景中仍存在两个问题。第一，数据具有可复制、可多次授权和难以排他占有的特点，直接讨论所有权转移容易与实际交易对象不匹配。第二，若只把区块链作为支付和登记工具，而不把质量评价、卖家信誉和争议证据纳入交易流程，则难以解决低质量数据获得同等曝光、买家维权依据不足等问题。本文因此把交易对象限定为指定时间窗口内 IoT 批次数据的访问权，并围绕访问权发放和交易证据设计合约逻辑。

## 2.2 Hyperledger Fabric 联盟链架构

Hyperledger Fabric 是面向联盟链场景的区块链框架，具有成员身份管理、通道隔离、链码执行和可插拔共识等特点。相较于开放式公链，Fabric 更适合企业、机构和监管方共同参与的数据流通场景。IoT 数据市场中的数据提供方、数据购买方、平台运营方和监管节点通常具备明确身份，交易过程也需要权限控制和审计能力，因此联盟链架构更符合该类系统的管理边界。

Fabric 的链码用于实现业务状态变更。链码不需要保存原始 IoT 文件，而是保存数据产品、订单、访问凭证哈希、质量分数和争议记录等结构化状态。背书、排序和提交流程保证交易结果在网络中形成一致账本。对于 Q-DataMarket，Fabric 层承担的是可信状态机角色：产品注册后形成可检索的链上目录，订单创建和卖家确认后形成押金约束，访问凭证发放后形成授权记录，交付确认和评价提交后更新交易结果，争议处理后保留裁决依据。

Fabric 研究和应用中常见的关注点包括吞吐量、交易延迟、背书策略、账本增长和链码执行开销。数据交易系统若要落地，不能只给出概念模型，还需要说明在原型实现中如何组织链码接口、如何进行链上链下分工、如何通过 Caliper 工作负载复现实验。本文在后续章节基于 Fabric 链码和 Node.js 服务层实现原型，并把性能评估限定为本地原型基准和 workload 配置推导所得结果，避免把未部署的大规模生产网络结果写成真实外部实验。

## 2.3 链上链下协同存储技术

IoT 批次数据通常具有规模大、更新频繁、字段类型多和时间序列连续性强等特点。若直接把 CSV 或 JSON 原始文件写入区块链，会导致账本快速膨胀、存储成本升高、隐私暴露风险增加，并降低交易处理效率。因此，链上链下协同存储成为区块链数据交易系统中的常用技术路线。其基本思想是：原始数据保存在链下对象存储或分布式文件系统中，链上保存数据哈希、元数据哈希、存储地址哈希、访问凭证哈希和业务状态。

链下存储负责承载大体量数据本体，链上账本负责保存不可篡改的摘要和交易状态。买家获得访问凭证后从链下服务下载数据，并用链上的 `dataHash` 验证文件完整性；若下载文件与链上哈希不一致，买家可提交证据触发争议。该方式既降低了链上存储压力，又保留了数据完整性校验和交易审计能力。对于访问权交易而言，链上记录授权凭证摘要比直接公开存储地址更安全，也更便于在凭证过期、重复访问或争议处理时进行责任界定。

已有链上链下协同研究通常关注文件哈希上链、存储地址索引和访问控制，但对交易后的质量评价、卖家信誉更新和仲裁证据结构关注不足。Q-DataMarket 在协同存储基础上进一步把质量分数、买家评分、争议惩罚和访问凭证摘要纳入链上状态，使链上记录不仅服务于完整性校验，也服务于数据排序、价格推荐、结算和仲裁。

## 2.4 IoT 数据市场与访问权交易

IoT 数据市场的核心目标是让设备产生的时序数据在不同主体之间流通和复用。与普通文件交易相比，IoT 批次数据具有明显的时空边界和设备来源属性。一个可交易的数据产品往往对应某一设备、某一时间窗口、某组传感器字段和某种质量水平。买家真正需要的通常不是永久占有原始数据所有权，而是在约定范围内获得读取、分析或验证数据的访问权。

将交易对象定义为访问权有三方面优势。第一，它符合数据可复制和可多次授权的特征，同一批数据可以在不改变原始持有关系的情况下授权给多个买家。第二，它便于设计限时访问凭证、访问审计和授权撤销策略。第三，它降低了所有权转移带来的法律和责任边界复杂性，使系统实现可以聚焦在授权、交付、验证和结算流程上。Q-DataMarket 因此把产品模型设计为 IoT 批次数据访问权，而非原始数据所有权。

现有数据市场研究常覆盖数据目录、撮合、定价和支付，但对访问凭证的链上记录、交付后的哈希校验、异常交易仲裁和质量驱动排序仍不充分。尤其在 IoT 场景中，低质量、缺失字段、异常值和时间戳不连续会直接影响买家后续分析效果。如果市场只提供目录和价格，买家很难在购买前评估数据可用性，卖家也缺少持续提升数据质量的激励。本文后续章节通过质量评分、卖家信誉和押金仲裁机制补足这一环节。

## 2.5 数据质量评价方法

数据质量评价通常从完整性、准确性、一致性、时效性、唯一性和可用性等维度展开。对于 IoT 批次数据，完整性反映字段缺失和记录缺失情况，一致性反映时间戳连续性、字段类型和状态码合法性，异常率反映传感器数值是否偏离合理范围，时效性反映数据产生时间与交易时间之间的距离。交易后买家评分则补充了客观指标难以覆盖的业务可用性反馈。

在数据交易场景中，质量评价不能只停留在展示层。如果质量分数不影响排序、价格和信誉，卖家缺少改进激励，买家也难以从市场机制中获得更高质量的匹配结果。Q-DataMarket 将质量评分与卖家信誉联动，把完整性、一致性、异常率、时效性、买家评分和争议惩罚统一纳入评价模型。质量分数参与检索排序，卖家信誉影响产品曝光，推荐价格随质量水平调整，从而形成质量驱动的数据交易机制。

已有研究中的质量评价模型有的偏重静态元数据，有的偏重人工评价，有的偏重特定领域指标。本文面向通用 IoT 批次数据，采用可由 CSV/JSON 样本和交易记录计算的指标，便于在原型系统中复现和测试。第 4 章将进一步给出评分维度、权重关系、信誉更新和价格调整策略，并与 `quality/score.js` 中的实现保持一致。

## 2.6 公平交易与争议仲裁机制

数据交易中的公平性问题主要来自交付不可见、数据质量不确定和双方行为不可完全信任。卖家可能上传与链上承诺不一致的数据，买家可能下载成功后拒绝确认，买家也可能发起虚假争议以试图退款。传统平台通常依赖中心化客服或人工仲裁处理这些问题，但裁决依据、处理过程和惩罚规则往往不够透明。

押金仲裁是一种可由智能合约实现的约束机制。买家下单时锁定价格和买方押金，卖家接受订单时锁定卖方押金；当交付成功并被确认时，系统向卖家结算并释放押金；当出现哈希不一致、超时不确认、虚假争议或凭证过期访问等异常场景时，合约根据链上证据执行退款、结算或扣罚。押金的作用不是增加交易复杂度，而是把恶意行为转化为可量化损失成本。

公平交易研究中常见的难点在于证据结构设计。如果证据无法被链上状态引用，仲裁仍然依赖平台主观判断。Q-DataMarket 通过 `dataHash`、`accessTokenHash`、`deadline` 和 `evidenceHash` 建立证据索引，使争议处理围绕可验证摘要和订单状态展开。第 5 章将结合链码接口说明产品注册、订单创建、卖家确认、访问凭证发放、交付确认、评分提交、争议开启和争议解决的完整状态流转。

## 2.7 现有研究不足

综上，现有研究为区块链数据交易、链上链下存储、质量评价和公平交易提供了基础，但面向 IoT 批次数据访问权交易仍存在以下不足。

第一，交易对象定义不够精确。一些研究将数据交易表述为所有权转移或文件买卖，未充分体现数据可复制、可多次授权和难以排他占有的特点。本文将交易对象限定为 IoT 批次数据访问权，更适合通过限时访问凭证和链上授权记录实现。

第二，质量评价与交易机制结合不足。部分研究提供质量标签或评分展示，但没有把评分结果用于排序、推荐价格和卖家信誉更新，难以形成持续激励。本文将质量评分嵌入检索排序、价格调整和信誉更新流程，并在实验中比较质量驱动方案与随机排序基线。

第三，公平仲裁缺少可执行的状态机和证据结构。仅记录交易结果不能解决数据哈希不一致、买家超时不确认、虚假争议和凭证过期访问等异常场景。本文通过押金锁定、访问凭证哈希、证据哈希和订单状态机实现链上仲裁。

第四，可复现实验支撑不足。部分方案停留在架构设计或理论分析，缺少与实现代码对应的 workload、样本数据和测试结果。本文基于 Hyperledger Fabric、Node.js 服务层、质量评分模块、IoT 样本生成脚本和 Hyperledger Caliper 配置组织实验，并明确区分本地原型基准、配置推导结果和未来可扩展的真实网络测试。

## 2.8 本章小结

本章梳理了区块链与智能合约、Hyperledger Fabric、链上链下协同存储、IoT 数据市场、数据质量评价和公平交易仲裁等相关技术与研究现状。分析表明，IoT 数据交易需要同时解决访问权建模、链上链下分工、质量激励、公平结算和可复现实验问题。Q-DataMarket 以访问权交易替代所有权转移，以质量评分和卖家信誉驱动排序与价格推荐，以押金和链上证据支撑争议仲裁，并基于 Fabric 原型和 Caliper workload 进行实验评估。后续章节将围绕总体设计、质量机制、仲裁机制、系统实现和实验结果展开论述。
