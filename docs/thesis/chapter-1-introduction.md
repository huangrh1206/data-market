# 第 1 章 绪论

## 1.1 研究背景

随着物联网、边缘计算和智能终端的快速发展，大量设备持续产生温度、湿度、压力、位置、电量、状态码等时序数据。这类数据不仅服务于设备运行监测，也可被复用于工业运维、城市治理、环境感知、风险预警和智能决策等场景。数据逐渐成为可流通、可复用、可增值的关键资源，围绕数据共享、数据开放和数据交易的研究与应用持续增长。

传统数据交易平台通常由中心化机构负责数据上架、撮合、授权、结算和争议处理。该模式实现简单，但存在明显局限。首先，平台方掌握交易流程和部分数据访问能力，数据提供方难以验证数据是否被越权使用或重复分发。其次，交易记录、授权记录和评价记录依赖平台维护，买卖双方对记录完整性和不可篡改性缺乏共同信任。再次，数据质量难以量化，低质量数据可能与高质量数据获得同等曝光，影响买方决策和市场效率。最后，发生下载失败、数据不一致或恶意争议时，中心化平台裁决缺乏透明证据支撑。

区块链具备分布式账本、不可篡改、可追溯和智能合约自动执行等特性，适合用于记录数据访问权交易过程。通过智能合约，系统可以自动执行订单状态流转、访问授权、结算、评价和仲裁逻辑。但 IoT 数据规模大、更新频繁，若将原始数据直接写入区块链，会导致存储成本过高、隐私风险增加和系统性能下降。因此，链上链下协同成为更合理的技术路线：原始数据存储在链下，区块链记录数据摘要、元数据哈希、订单状态、授权凭证和争议证据。

## 1.2 研究意义

本文研究基于区块链的 IoT 数据访问权交易系统，具有以下意义。

第一，在数据流通层面，本文将交易对象定义为指定设备、指定时间窗口内 IoT 批次数据的访问权，而不是直接转移原始数据所有权。该定义符合数据可复制、可多次授权、难以独占转移的特点，有助于构建更清晰的数据交易模型。

第二，在可信交易层面，本文通过区块链记录数据摘要、订单状态、访问凭证摘要和争议证据，使买卖双方可以共同验证交易过程。该机制有助于降低中心化平台带来的信任风险。

第三，在数据质量层面，本文设计质量评分和卖家信誉联合驱动机制，使数据质量不只作为展示信息，而是参与检索排序、推荐价格和信誉更新。该机制能够激励数据提供方提升数据质量。

第四，在公平交易层面，本文设计押金锁定和链上证据仲裁机制，约束卖家上传不一致数据、买家超时不确认、买家虚假争议等恶意行为，提高交易公平性。

## 1.3 研究内容

本文围绕通用 IoT 批次数据访问权交易，开展以下研究。

1. 设计链上链下协同的数据访问权交易模型。系统将 IoT CSV/JSON 批次数据保存到链下对象存储，将数据哈希、元数据哈希、价格、订单状态、访问凭证哈希和争议证据保存到区块链。

2. 设计质量驱动的数据交易机制。系统从完整性、一致性、异常率、时效性和买家评价五个维度计算数据质量评分，并结合卖家信誉影响数据产品排序和推荐价格。

3. 设计基于押金仲裁的公平交易机制。系统通过订单状态机、押金锁定、访问凭证和证据哈希，对哈希不一致、超时不确认、虚假争议和凭证过期访问等异常场景执行退款、结算或扣罚。

4. 实现 Q-DataMarket 原型系统。系统基于 Hyperledger Fabric 实现链码，基于 Node.js 实现服务层，基于本地对象存储或 MinIO 保存链下数据，并使用 Hyperledger Caliper 设计性能测试工作负载。

5. 完成实验与性能评估。本文从基础性能、质量机制有效性和仲裁机制有效性三个角度评价系统，指标包括 TPS、平均延迟、P95 延迟、成功率、高质量数据成交占比、争议率和平均仲裁延迟。

## 1.4 主要贡献

本文主要贡献如下。

1. 提出一种链上链下协同的 IoT 批次数据访问权交易模型。该模型避免原始数据直接上链，降低链上存储压力，同时保留交易可验证、可追溯和可审计能力。

2. 提出质量评分与卖家信誉联合驱动的数据交易机制。该机制将数据完整性、一致性、异常率、时效性和买家评价纳入统一评分，并将评分结果用于排序、推荐价格和信誉更新。

3. 提出基于押金与链上证据的公平仲裁机制。该机制使用订单状态机和证据哈希约束恶意交易行为，支持退款、结算和扣罚。

4. 实现基于 Hyperledger Fabric 的原型系统，并使用 Hyperledger Caliper 进行性能评估。实验设计覆盖数据发布、下单购买、访问授权、确认结算、质量评价和争议仲裁等核心流程。
